Top Applications of Redis Database:

    	Caching (Database & Content): Stores frequently accessed data (e.g., query results, HTML pages) to reduce latency and alleviate load on primary databases.
    
	Session Management: Stores user session data (logins, shopping carts) across distributed web apps for fast retrieval.
    
	Real-Time Leaderboards: Utilizes Sorted Sets to maintain, update, and retrieve real-time rankings instantly, commonly used in gaming.
    	
	Pub/Sub Messaging & Queues: Enables real-time chat, notifications, and task queuing (e.g., background jobs) with high throughput.
    
	Real-Time Analytics & Personalization: Processes large volumes of streaming data instantly for live metrics, product recommendations, and behavioral targeting.
    
	Geospatial Applications: Stores and queries location-based information, such as finding nearby drivers or places.
    
	Rate Limiting: Controls API usage by counting requests per user/IP over specific time frames.

Sorted Sets - mix of a hash and a set, no keys and values, there are members and scores
members are unique, scores don't have to be unique, scores are sorted from least to greatest

- ZADD products 45 monitor
- ZSCORE products monitor
- ZREM products monitor
- ZSCORE products monitor
- DEL products

- ZADD products 45 cpu
- ZADD products 10 cupboard
- ZADD products 55 power

- ZCARD products // cardinality of products set 
- ZCOUNT products 0 50 // count products with score between 0 and 50 including 0 and 50 
- ZCOUNT products 0 (55  // greater than 0 and less than 55
- ZCOUNT products -inf +inf
- ZPOPMIN - remove lowest score pairs
- ZPOPMAX - remove highest score pairs
- ZPOPMIN products 2 //remove lowest 2 
- ZPOPMAX products 2
- ZINCRBY products 15 keyboard
- ZINCRBY products 15 cpu
- ZSCORE products cpu
- ZRANGE products 1 2 WITHSCORES // 1 and 2 are indexes, including, withscores gives scores also
- ZRANGE products 0 48 BYSCORE WITHSCORES // 0 minimum score and 48 maximum score
- ZRANGE products 0 50 BYSCORE WITHSCORES 
- ZRANGE products 0 (55 BYSCORE WITHSCORES // less than 55 not including
- ZRANGE products -inf +inf BYSCORE WITHSCORES 
- ZRANGE products 1 2 REV LIMIT 1 2 // set product, 1 min index, 2 max index, reverse order first, skip 1st element, then give next two
- ZRANGE products 0 55 BYSCORE LIMIT 1 1

Lists
- LPUSH temps 25 // push element 25 to left side of list
- RPUSH temps 27 // push element 27 to right side of list
- LLEN temps // get the number of elements in the list
- LINDEX temps -1 // get the value stored at the provided index
- LINDEX temps 0
- LRANGE temps 0 3 // all elements in list from index 0 to 3, including
- LPOS temps 25 RANK 2 COUNT 2 MAXLEN 10 // 2nd instance of 25, retrieve the index of the first 2 matches, only search for first 10 elements 
- LPOP temps 2 // remove first 2 elements
- RPOP temps 2 // remove last 2 elements
- LSET temps 2 32 // go to index 2 and update to 32
- LSET temps -1 35 // change last element to 35
- LTRIM temps 2 5 // only keep elements from index 2 to 5 including
- LINSERT temps BEFORE 25 15 // insert 15 before 25
- LINSERT temps AFTER 30 15 // insert 30 after 15
- LREM temps -2 25 // move from right to left, remove 2 instances of 25
- LREM temps 1 25 // moving from left to right remove 1 instance 0f 25
- LREM temps 0 25 // 0 means remove all copies of 25
- LRANGE temps 0 -1 // print all elements

List use cases- append-only/prepend-only data(temperature readings/stock values)
- when to fetch last/first N values of something
- no sort order besides the order it was inserted

Should not used
- you need to apply some filtering criteria, your data is sorted by some attribute
